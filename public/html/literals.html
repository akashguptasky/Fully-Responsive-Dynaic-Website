Literals



Any constant value which can be assing to the variable is called "Literals".
Example : int x = 10;
Where,
int ==>> data type or keyword
x ==>> name of varaible or Identifier
10 ==>> constant value or Literal

Integral Literals


For Integral data type (byte, short, int, long) we can specity literal value in the following base :-
  1. Decimal

    2. Allowed digit are 0 to 9.
    Example 1: int x = 10; ✔️
    Example 2: int x = 2000; ✔️
    Example 3: int x =899; ✔️

  2. Octal

    3. Allowed digit are 0 to 7 and literal value shoud be prefix with zero(0).
    Example 1: int x = 010; ✔️
    Example 2: int x = 040; ✔️
    Example 3: int x = 0777; ✔️
    Example 4: int x = 0786; ❌ [integer number too large as we know that the range of octal is 0 to 7 but here we use 8 which is out of range.]

  3. Hexa Decimal

    4. Allowed digit are 0 to 9, A to F, a to f, for extra digit digits we can use both lowercase and uppercase character this is one of very few areas where java is not case sensitive. The literal value shold be prefix with the 0x / 0X.
    Example 1: int x = 0x10; ✔️
    Example 2: int x = 0xface; ✔️
    Example 3: int x = 0XFACE; ✔️
    Example 4: int x = BEER; ❌ [out of range in hexa (a to f)]

  4. Binary

    5. For Integeral data type until 1.6 Version. we can specitfy literal value in the following base.
    1. Decimal form
    2. Octal form
    3. Hexa decimal form
    but 1.7 version onwards, we can specity literal value even in binary form also.

    • Allow digits are 0 and 1.
    • Litiral value should be prefix with the 0b/0B
    Example 1: int x = 0b1111; ✔️
    Output: 15




Floating Point Literals

By default every floating point literals is "double type". Hence we can't directly to the float variable but we can specify floating point literals as float type by sufix with the "f" or "F".
Example:
1. float a =123.456; ❌ [Compiler Error: incompatible types: possibly lossy Conversion from double to float.   Found: double    Required: flaot
2. float a = 123.456f; (here we are using sufix 'f');
3. double a = 123.456;


We can speciy explacitly floating point literals as a double type by sufix with the 'd' or 'D' ofcourse this conversion is not required.


Example: double d = 1223.456D;
double d = 1223.456d;
float a = 134.242D; ❌ [Compiler Error: incompatible types: possibly lossy Conversion from double to float.   Found: double    Required: flaot


we can specify floating point literals only in decimal form and we can't specify in octa and Hexa-Decimal forms.
Example:
double d = 123.456;
double d = 0123.456; [Notes: its a decimal literal but ot octal literal.]
double d = 0x123.456; [Notes: Malformed folating point literal.]


we can assing integral literal directly to floating point variable and that itergral literals can be specified either int or decimal or octal or hexa-decimal form.
Example 1: double d = 0765; ✔️ [Octal form and in the range]
Example 2: double d = 0786; ✔️ [Octal form but out the range because as we know that the range in octal in 0 to 7 so here we will get compile time error :interger no too large.]
Example 3: double d = 0xFace; ✔️ [Hexa form , in the range]
Example 4: double d = 0xface.0; ✔️ [Hexa form , but floating point is not allowed so it's wrong]
Example 5: double d = 10; ✔️
Example 6: double d = 0777;✔️ [octal form , in the range ]

We can't assign floating point literals to integral types.
Example 1: double d = 10; ✔️
Example 2: double d = 10.0; ✔️ [Compiler Error : incompatible types: possibly lossy conversion from double to int.     found: double     Required: int]

We can specify floating point literals even in Exponential form (Sciencetific notation).
Example 1: double d = 1.2e3; ✔️ [Output: 1200.0]
Example 2: float f = 1.2e3; ❌ [Compiler Error: incompatible types : Possible loosy conversion from double to int]
Example 3: float f = 1.2e3f;

Note: Sciencetific Notation is a way of expressing numbers that are too large or too small to be conveniently written in decimal form or statndard index form or statndard form in the uk.
