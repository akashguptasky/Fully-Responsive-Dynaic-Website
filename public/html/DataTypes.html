DataTypes



  1. In Java every variable and every assignments has some type. Each and every data type is clearly defined in java. Every assingment should be check by compiler for type compatibility.
  2. because of above reasons we can conclude java language is "Strongly Typed" programming language.
  3. java is not consider as pure object oriented programming language because several boop feature are not satisfied by java (like Operator overloading, multiple inherirtance etc.).
  4. we are depending on premitives data types which are non-objects.




  5. Except boolean and char remaining data type are consider as "signed data types" because we can represent both positive ("+ve") and negative ("-ve") number


About Byte


  • Size: 1byte (8bits)
  • MAX_VALUE:+127
  • MIN_VALUE:-128
  • Range: -128 to 127




  • The most significant bit acts as sign bit. "0" means "+ve" number and "1" means "-ve" number.
  • "+ve" numbers will be represented directly in the memory whereas "-ve" numbers represented in 2's complement form.

    • ➡️Example 1:
    byte b=10;✔️
    byte b=130;❌
    [CE: possible loss of precision]
    found: int
    required:byte

    ➡️Example 2:

    byte b=127;✔️
    byte b=128;❌
    [CE: possible loss of precision]
    found: int
    required:byte

    ➡️Example 3:
    byte b=10.5;     ❌ [CE: possible loss of precision]
    byte b=true;      ❌ [CE: incompatible types]
    byte b="ashok";❌[CE: incompatible types]
    found: java.lang.String
    required:byte

    Note: byte data type is best suitable if we are handling data in terms of streams from the file or from the network.

About long data type



Sometimes int may not enough to hold big values then we should go for long data type.
Example:
To hold the no of characters present in a big file int may not enough hence the return type of length() method is long.


  • Size: 8 bytes (64 bits)
  • Range: -263 to 263-1


Note: All the above data types (byte, short, int, long) main differ representing integral values. If we want to represent floating point value then we should go for floating point data types.

About flaot vs double



float: If we want 5 to 6 decimal places of accuracy then we should go for float. float follow single precision. The size of float is 4 bytes.
double:if we want 14 to 15 decimal places of accuracy then we should go for double.double follow double precision. The size of double is 8 bytes.

About char



Old language (like c, c++) are ASCII code based and the number of allowed different ASCII code character are <= 256. To represent these 256 character 8 bits are enough hence the size of char in old languages is 1 byte.
But java is "UNICODE" based and the number of different uniccode character greather than 256 and less then 65,536. To represent these many character "8 bits (1 bytes) is not enough" compulsory we sshould go for 16 bit's. Hence the size of char in java is 2 bytes.

  • Size: 2 bytes
  • Range: 0 to 65,535
